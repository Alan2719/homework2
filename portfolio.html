The purpose of this activity was to create a password generator. The user is able to decide which type of characters the password will have (uppercase letters, lowercase letters, numbers and special characters). The user is also able to decide the length of the password between 7 to 120 characters.

In this activity, a general culture quiz was created. The player only has 20 seconds to answer 5 questions. Every time a question is correct , the player earns 5 extra seconds. If the answer is incorrect, the player looses 5 seconds. The final score is the seconds left in the timer. You are able to save your score by typing your name.

The aim of this activity was to create a day planner to use it in your daily work. With this planner you are able to schedule your to do list per hour as well as modify or erase them once you have finished. The colors indicate if you are on time or is to late to late to perform the task.

The purpose of this activity was to create a weather dashboard using the OpenWeather API. By searching the city, you are able to view the current temperature, wind speed, UV index, and Humidity. Also, you are able to visualize a 5 day forecast. The cities you have already searched are stored in case you want to check them again.

In our first project, my team and I created the Band Tracker website. In this website you are able to create an account and login to the site. The principal purpose of this site is to look for your favorite bands and have some information of them like the monthly listeners, playcount and the upcoming events. You are able to follow or unfollow your favorite artist. Finally, you can be redirected to another site to buy tickets from the next concert.

The aim of this application is to create an organization chart in which the user can create three types of positions, a manager, an engineer and an intern. Using the npm package inquirer, the user is asked fills the worker profile with different information depending of the position. At the end, an html file is created with the organization chart.

The purpose of this task was to create a note taker. This is the first activity in which a server was used as well as NodeJs, and express to develop the backend. The application lets the user to save notes. When a note is no longer useful, you can delete it. The deployment of this application was made in Heroku.

In this task a readme file generator was created. When the application is running, the user needs to insert the title of the readme file, description, installation, usage, license, and some other questions. In this case, the package inquirer is used. Also, promises are used to the application run correctly.

In this application, a company database was created. The database contains 3 tables, department, position, and employee. Using inquirer and MySQL, the user is able to create new departments, roles, and insert new employees to the database. The application is also able to display just the information the user want to see intercacting with the database.

In this application, the user is able to introduce their favorite burgers, save in the database, and view them in the frontend. The user is able to eat that burger with a button created when the name of the burger is displayed in the frontend. Once the burger, is eaten, it is displayed in the right side of the screen. For this application, I used express, handlebars, NodeJs, and MySQL. The application was desployed in Heroku.

In our project 2, my team and I created an application where different kind of services could be post. A user, offering a service, can post their contact info. Another user, looking for a service, can look in the catalogue all the services available and contact the best qualified person to do the job. Inside the website, users are able to create a profile, and new categories. People can also leave comments about the service they hired for future users.
